本システムで使用するクラウドデータベース（Supabase PostgreSQL）、およびオフライン一時保存用のクライアントサイドデータベース（IndexedDB）の定義について記述します。

---

## 1. クラウドデータベース（Supabase PostgreSQL）定義

### 1.1 テーブル一覧
| 物理テーブル名 | 論理テーブル名 | 用途 |
| :--- | :--- | :--- |
| **`departments_master`** | 学科マスタテーブル | 学科名・修業年限を管理するマスタ。入力フォームの選択肢の根拠となる。 |
| **`department_classes`** | クラスマスタテーブル | 学科・学年ごとのクラス一覧を管理するマスタ。 |
| **`users_cache`** | 学生キャッシュテーブル | 利用者の入力負荷を減らすため、一度手動入力された情報を学籍番号キーで保存しておくテーブル。利用統計・ランキング表示にも利用する。 |
| **`usage_logs`** | 利用ログテーブル | ジムの利用記録（チェックイン・チェックアウト履歴）を保存するテーブル。利用時間・自動チェックアウト・管理者承認状態を保持する。 |
| **`notifications`** | 通知テーブル | 利用ログに紐づく通知メッセージと承認状態を管理する。 |
| **`annual_grade_promotions`** | 年度更新履歴テーブル | 4月の学生キャッシュ自動進級が同一年度に複数回実行されないよう、実行年度と件数を保持する。 |

### 1.2 `departments_master` テーブル定義
* **役割**: 学科名と修業年限（何年制か）を管理する。入力フォームの学科プルダウン・学年プルダウンの選択肢を提供する。削除時は物理削除せず、ゴミ箱へ移動する。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE departments_master (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      name VARCHAR(100) NOT NULL UNIQUE,
      years_count SMALLINT NOT NULL DEFAULT 2,
      sort_order SMALLINT NOT NULL DEFAULT 0,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      updated_at TIMESTAMPTZ DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL
  );

  CREATE TRIGGER update_departments_master_modtime
      BEFORE UPDATE ON departments_master
      FOR EACH ROW
      EXECUTE PROCEDURE update_modified_column();

  CREATE INDEX idx_departments_master_active
      ON departments_master(sort_order, id)
      WHERE deleted_at IS NULL;
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | 学科ID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `name` | 学科名 | VARCHAR(100) | NOT NULL, UNIQUE | - | 学科名（例: 建築学科）。一意であること。 |
| `years_count` | 修業年限 | SMALLINT | NOT NULL | 2 | 何年制か（2年制なら2、4年制なら4）。学年プルダウンの選択肢絞り込みに使用。 |
| `sort_order` | 表示順 | SMALLINT | NOT NULL | 0 | プルダウン内での表示順序。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | レコード作成日時。 |
| `updated_at` | 更新日時 | TIMESTAMPTZ | - | NOW() | レコード最終更新日時。 |
| `deleted_at` | 論理削除日時 | TIMESTAMPTZ | - | NULL | 管理者が削除した日時。NULLの場合は有効な学科。30日以内はゴミ箱から復元可能。30日経過または完全削除で物理削除。 |

---

### 1.3 `department_classes` テーブル定義
* **役割**: 学科・学年の組み合わせごとに存在するクラス名を管理する。入力フォームのクラス選択肢を提供する。所属学科が削除された場合は同時にゴミ箱へ移動する。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE department_classes (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      department_id BIGINT NOT NULL REFERENCES departments_master(id) ON DELETE CASCADE,
      grade SMALLINT NOT NULL,
      class_name VARCHAR(50) NOT NULL,
      sort_order SMALLINT NOT NULL DEFAULT 0,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL,
      UNIQUE(department_id, grade, class_name)
  );

  CREATE INDEX idx_department_classes_dept_grade
      ON department_classes(department_id, grade);

  CREATE INDEX idx_department_classes_active
      ON department_classes(department_id, grade, sort_order, id)
      WHERE deleted_at IS NULL;
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | クラスID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `department_id` | 学科ID | BIGINT | NOT NULL, FK | - | `departments_master.id` への外部キー。 |
| `grade` | 学年 | SMALLINT | NOT NULL | - | 対象学年（1〜N）。 |
| `class_name` | クラス名 | VARCHAR(50) | NOT NULL | - | クラス名（例: A組）。 |
| `sort_order` | 表示順 | SMALLINT | NOT NULL | 0 | プルダウン内での表示順序。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | レコード作成日時。 |
| `deleted_at` | 論理削除日時 | TIMESTAMPTZ | - | NULL | 管理者が削除した日時。NULLの場合は有効なクラス。所属学科が論理削除された場合は同時に論理削除され、学科復元時にまとめて復元される。 |

### 1.4 `users_cache` テーブル定義
* **役割**: 学籍番号を入力した際に、過去の入力情報（氏名やクラス）を自動補完するためのキャッシュ。同一の有効な学籍番号は1件のみ保持し、初回登録フォームから既存学籍番号が送信された場合でも重複レコードを作成しない。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE users_cache (
      student_id VARCHAR(50) PRIMARY KEY,
      name VARCHAR(100) NOT NULL,
      department VARCHAR(100) NOT NULL,
      grade VARCHAR(20) NOT NULL,
      class_name VARCHAR(50) NOT NULL,
      is_staff BOOLEAN NOT NULL DEFAULT FALSE,
      user_type VARCHAR(20) NOT NULL DEFAULT 'student',
      total_usage_minutes INTEGER NOT NULL DEFAULT 0,
      monthly_usage_minutes INTEGER NOT NULL DEFAULT 0,
      consecutive_days INTEGER NOT NULL DEFAULT 0,
      last_used_date DATE,
      registered_at TIMESTAMPTZ DEFAULT NOW(),
      created_at TIMESTAMPTZ DEFAULT NOW(),
      updated_at TIMESTAMPTZ DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL
  );

  -- 更新日時自動更新用のトリガー
  CREATE OR REPLACE FUNCTION update_modified_column()
  RETURNS TRIGGER AS $$
  BEGIN
      NEW.updated_at = NOW();
      RETURN NEW;
  END;
  $$ language 'plpgsql';

  CREATE TRIGGER update_users_cache_modtime
      BEFORE UPDATE ON users_cache
      FOR EACH ROW
      EXECUTE PROCEDURE update_modified_column();

  -- 論理削除されていないレコードへの高速アクセス用インデックス
  CREATE INDEX idx_users_cache_active
      ON users_cache(student_id)
      WHERE deleted_at IS NULL;
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `student_id` | 学籍番号 | VARCHAR(50) | PRIMARY KEY | - | 学生の学籍番号または教職員番号。学籍番号は `^\d{7}$`、教職員番号は `^T\d{3}$` のいずれかを満たす。アプリケーション側でのバリデーションと合わせて運用する。 |
| `name` | 氏名 | VARCHAR(100) | NOT NULL | - | 学生の氏名。 |
| `department` | 学科 | VARCHAR(100) | NOT NULL | - | 所属学科。 |
| `grade` | 学年 | VARCHAR(20) | NOT NULL | - | 学年。 |
| `class_name` | クラス名 | VARCHAR(50) | NOT NULL | - | クラス名。 |
| `is_staff` | 教職員フラグ | BOOLEAN | NOT NULL | FALSE | 教職員の場合TRUE。学科・学年入力を省略するために参照する。 |
| `user_type` | 利用者種別 | VARCHAR(20) | NOT NULL | `student` | `student` / `staff` を保持し、画面表示や判定に利用する。 |
| `total_usage_minutes` | 累計利用時間 | INTEGER | NOT NULL | 0 | 累計利用時間（分）。 |
| `monthly_usage_minutes` | 月間利用時間 | INTEGER | NOT NULL | 0 | 月間利用時間（分）。ランキング表示に利用する。 |
| `consecutive_days` | 連続利用日数 | INTEGER | NOT NULL | 0 | 連続利用日数。 |
| `last_used_date` | 最終利用日 | DATE | - | NULL | 最新の利用日。 |
| `registered_at` | 登録日時 | TIMESTAMPTZ | - | NOW() | 初回登録日時。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | レコードが最初に作成された日時。 |
| `updated_at` | 更新日時 | TIMESTAMPTZ | - | NOW() | レコードが最後に更新された日時。 |
| `deleted_at` | 論理削除日時 | TIMESTAMPTZ | - | NULL | 管理者が削除した日時。NULLの場合は有効なレコード。30日経過後に物理削除。 |

### 1.5 `usage_logs` テーブル定義
* **役割**: ジムの利用記録（チェックイン・チェックアウト）。オンライン時は直接インサートされ、オフライン時は復帰後にバルクインサートされます。
* **補足**: `users_cache.student_id` / `usage_logs.student_id` には、学籍番号 `^\d{7}$` または教職員番号 `^T\d{3}$` のいずれかであることを保証する CHECK 制約またはアプリケーション層のバリデーションによる整合性ルールを適用する。実装では共通の ID フォーマット定数に集約して管理する。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE usage_logs (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      student_id VARCHAR(50) NOT NULL,
      name VARCHAR(100) NOT NULL,
      department VARCHAR(100) NOT NULL,
      grade VARCHAR(20) NOT NULL,
      class_name VARCHAR(50) NOT NULL,
      user_type VARCHAR(20) NOT NULL DEFAULT 'student',
      checked_in_at TIMESTAMPTZ NOT NULL,
      checked_out_at TIMESTAMPTZ DEFAULT NULL,
      auto_checked_out BOOLEAN NOT NULL DEFAULT FALSE,
      usage_duration_minutes INTEGER DEFAULT NULL,
      is_adjusted BOOLEAN NOT NULL DEFAULT FALSE,
      is_notified BOOLEAN NOT NULL DEFAULT FALSE,
      admin_confirmed BOOLEAN NOT NULL DEFAULT FALSE,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL
  );

  -- 検索高速化のためのインデックス
  CREATE INDEX idx_usage_logs_student_id ON usage_logs(student_id);
  CREATE INDEX idx_usage_logs_checked_in_at ON usage_logs(checked_in_at);

  -- 在室中（チェックアウト未記録）レコードの高速検索用インデックス
  CREATE INDEX idx_usage_logs_checkin_active
      ON usage_logs(student_id, checked_in_at)
      WHERE checked_out_at IS NULL AND deleted_at IS NULL;

  -- 論理削除されていないレコードへのインデックス（運用後の件数増加に応じて追加を検討）
  -- CREATE INDEX idx_usage_logs_active
  --     ON usage_logs(checked_in_at)
  --     WHERE deleted_at IS NULL;
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | ログID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `student_id` | 学籍番号 | VARCHAR(50) | NOT NULL | - | 入室した学生の学籍番号または教職員番号。学籍番号は `^\d{7}$`、教職員番号は `^T\d{3}$` のいずれかを満たす。 |
| `name` | 氏名 | VARCHAR(100) | NOT NULL | - | 入室した学生の氏名。 |
| `department` | 学科 | VARCHAR(100) | NOT NULL | - | 入室した学生の学科。 |
| `grade` | 学年 | VARCHAR(20) | NOT NULL | - | 入室した学生の学年。 |
| `class_name` | クラス名 | VARCHAR(50) | NOT NULL | - | 入室した学生のクラス名。 |
| `user_type` | 利用者種別 | VARCHAR(20) | NOT NULL | `student` | `student` / `staff` を保持する。 |
| `checked_in_at` | チェックイン時間 | TIMESTAMPTZ | NOT NULL | - | 利用者が「チェックイン」を押した（またはスキャンが成功した）時間。 |
| `checked_out_at` | チェックアウト時間 | TIMESTAMPTZ | - | NULL | 退室時間。NULLの場合は在室中または自動チェックアウト未処理。チェックイン後15時間経過で自動的にセットされる。 |
| `auto_checked_out` | 自動退室フラグ | BOOLEAN | NOT NULL | FALSE | 15時間超過によりシステムが自動的に退室処理した場合にTRUE。 |
| `usage_duration_minutes` | 利用時間 | INTEGER | - | NULL | チェックアウト時に算出される利用時間（分）。自動補正時は一律30分として記録する。 |
| `is_adjusted` | 自動補正フラグ | BOOLEAN | NOT NULL | FALSE | 15時間超過時に一律30分として補正された利用ログである場合にTRUE。 |
| `is_notified` | 通知済みフラグ | BOOLEAN | NOT NULL | FALSE | ユーザーへ通知を表示済みの場合にTRUE。初期値はFALSE。 |
| `admin_confirmed` | 管理者承認フラグ | BOOLEAN | NOT NULL | FALSE | 管理画面上で承認済みかどうか。 |
| `created_at` | レコード作成日時 | TIMESTAMPTZ | - | NOW() | データベースに書き込まれた日時（サーバー時間）。 |
| `deleted_at` | 論理削除日時 | TIMESTAMPTZ | - | NULL | 管理者が削除した日時。NULLの場合は有効なレコード。30日経過後に物理削除。 |

### 1.6 `notifications` テーブル定義
* **役割**: 利用ログに紐づく通知メッセージを管理し、管理画面の通知ベルで表示する。`usage_logs.is_adjusted` / `usage_logs.is_notified` は利用者への自動補正通知状態を管理し、`notifications` は通知一覧・ベル表示の履歴を保持する。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE notifications (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      type VARCHAR(50) NOT NULL,
      usage_log_id BIGINT REFERENCES usage_logs(id) ON DELETE SET NULL,
      student_number TEXT,
      department TEXT,
      grade TEXT,
      name TEXT,
      message TEXT NOT NULL,
      is_read BOOLEAN NOT NULL DEFAULT FALSE,
      is_acknowledged BOOLEAN NOT NULL DEFAULT FALSE,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      read_at TIMESTAMPTZ DEFAULT NULL
  );
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | 通知ID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `type` | 通知種別 | VARCHAR(50) | NOT NULL | - | `auto_checkout` など通知の種類。 |
| `usage_log_id` | 利用ログID | BIGINT | FK | - | 関連する `usage_logs.id`。削除時は `NULL` にする。 |
| `student_number` | 学籍番号 | TEXT | - | - | 通知対象の学籍番号。 |
| `department` | 学科 | TEXT | - | - | 通知対象の学科。 |
| `grade` | 学年 | TEXT | - | - | 通知対象の学年。 |
| `name` | 氏名 | TEXT | - | - | 通知対象者の氏名。 |
| `message` | メッセージ | TEXT | NOT NULL | - | 管理画面に表示する通知本文。 |
| `is_read` | 既読フラグ | BOOLEAN | NOT NULL | FALSE | 管理画面の通知一覧で既読かどうか。 |
| `is_acknowledged` | 承認フラグ | BOOLEAN | NOT NULL | FALSE | 管理者が通知を確認・承認したかどうか。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | 通知が作成された日時。 |
| `read_at` | 既読日時 | TIMESTAMPTZ | - | NULL | 通知が既読になった日時。 |

### 1.7 `annual_grade_promotions` テーブル定義
* **役割**: 毎年4月に実行する学生キャッシュの自動進級・最終学年論理削除処理について、対象年度ごとの実行履歴を保持する。同一年度の二重実行を防止するため、`school_year` を主キーとする。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE annual_grade_promotions (
      school_year INTEGER PRIMARY KEY,
      promoted_count INTEGER NOT NULL DEFAULT 0,
      deleted_count INTEGER NOT NULL DEFAULT 0,
      executed_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `school_year` | 年度 | INTEGER | PRIMARY KEY | - | 4月に開始する年度。例: 2026年4月実行分は `2026`。 |
| `promoted_count` | 進級件数 | INTEGER | NOT NULL | 0 | 学年を1つ繰り上げた学生キャッシュ件数。 |
| `deleted_count` | 論理削除件数 | INTEGER | NOT NULL | 0 | 最終学年として論理削除した学生キャッシュ件数。 |
| `executed_at` | 実行日時 | TIMESTAMPTZ | NOT NULL | NOW() | 年度更新処理を完了した日時。 |

## 2. クライアントサイドデータベース（IndexedDB）定義

設置PCのブラウザがオフライン（インターネット未接続）状態のとき、一時的に利用ログを蓄積しておくためのローカル定義です。

* **データベース名**: `JimReserveOfflineStore`
* **ストア（テーブル）名**: `offline_logs`
* **キーパス (KeyPath)**: `id` (Auto Increment)

#### オブジェクトストア構造 (`offline_logs`)
| プロパティ名 | データ型 | 説明 |
| :--- | :--- | :--- |
| `id` | Number | ローカル自動連番（主キー） |
| `action` | String | `"check_in"` または `"check_out"` |
| `student_id` | String | 学籍番号 |
| `name` | String | 氏名 |
| `department` | String | 学科 |
| `grade` | String | 学年 |
| `class_name` | String | クラス名 |
| `is_staff` | Boolean | 教職員フラグ |
| `checked_in_at` | String (ISO 8601) | 利用者がチェックインした日時 |
| `checked_out_at` | String (ISO 8601) | チェックアウト時刻 |
| `log_id` | Number | 同期済み `usage_logs.id` （任意） |
