本システムで使用するクラウドデータベース（Supabase PostgreSQL）、およびオフライン一時保存用のクライアントサイドデータベース（IndexedDB）の定義について記述します。

## 1. クラウドデータベース（Supabase PostgreSQL）定義

### 1.1 テーブル一覧
| 物理テーブル名 | 論理テーブル名 | 用途 |
| :--- | :--- | :--- |
| **`departments_master`** | 学科マスタテーブル | 学科名・修業年限を管理するマスタ。入力フォームの選択肢の根拠となる。 |
| **`department_classes`** | クラスマスタテーブル | 学科・学年ごとのクラス一覧を管理するマスタ。 |
| **`users_cache`** | 学生キャッシュテーブル | 利用者の入力負荷を減らすため、一度手動入力された情報を学籍番号キーで保存しておくテーブル。 |
| **`usage_logs`** | 利用ログテーブル | ジムの利用記録（チェックイン・チェックアウト履歴）を保存するテーブル。 |

### 1.2 `departments_master` テーブル定義
* **役割**: 学科名と修業年限（何年制か）を管理する。入力フォームの学科プルダウン・学年プルダウンの選択肢を提供する。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE departments_master (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      name VARCHAR(100) NOT NULL UNIQUE,
      years_count SMALLINT NOT NULL DEFAULT 2,
      sort_order SMALLINT NOT NULL DEFAULT 0,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      updated_at TIMESTAMPTZ DEFAULT NOW()
  );

  CREATE TRIGGER update_departments_master_modtime
      BEFORE UPDATE ON departments_master
      FOR EACH ROW
      EXECUTE PROCEDURE update_modified_column();
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | 学科ID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `name` | 学科名 | VARCHAR(100) | NOT NULL, UNIQUE | - | 学科名（例: 建築学科）。一意であること。 |
| `years_count` | 修業年限 | SMALLINT | NOT NULL | 2 | 何年制か（2年制なら2、4年制なら4）。学年プルダウンの選択肢絞り込みに使用。 |
| `sort_order` | 表示順 | SMALLINT | NOT NULL | 0 | プルダウン内での表示順序。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | レコード作成日時。 |
| `updated_at` | 更新日時 | TIMESTAMPTZ | - | NOW() | レコード最終更新日時。 |

---

### 1.3 `department_classes` テーブル定義
* **役割**: 学科・学年の組み合わせごとに存在するクラス名を管理する。入力フォームのクラス選択肢を提供する。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE department_classes (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      department_id BIGINT NOT NULL REFERENCES departments_master(id) ON DELETE CASCADE,
      grade SMALLINT NOT NULL,
      class_name VARCHAR(50) NOT NULL,
      sort_order SMALLINT NOT NULL DEFAULT 0,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      UNIQUE(department_id, grade, class_name)
  );

  CREATE INDEX idx_department_classes_dept_grade
      ON department_classes(department_id, grade);
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | クラスID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `department_id` | 学科ID | BIGINT | NOT NULL, FK | - | `departments_master.id` への外部キー。 |
| `grade` | 学年 | SMALLINT | NOT NULL | - | 対象学年（1〜N）。 |
| `class_name` | クラス名 | VARCHAR(50) | NOT NULL | - | クラス名（例: A組）。 |
| `sort_order` | 表示順 | SMALLINT | NOT NULL | 0 | プルダウン内での表示順序。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | レコード作成日時。 |

### 1.4 `users_cache` テーブル定義
* **役割**: 学籍番号を入力した際に、過去の入力情報（氏名やクラス）を自動補完するためのキャッシュ。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE users_cache (
      student_id VARCHAR(50) PRIMARY KEY,
      name VARCHAR(100) NOT NULL,
      department VARCHAR(100) NOT NULL,
      grade VARCHAR(20) NOT NULL,
      class_name VARCHAR(50) NOT NULL,
      is_staff BOOLEAN NOT NULL DEFAULT FALSE,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      updated_at TIMESTAMPTZ DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL
  );

  -- 更新日時自動更新用のトリガー
  CREATE OR REPLACE FUNCTION update_modified_column()
  RETURNS TRIGGER AS $$
  BEGIN
      NEW.updated_at = NOW();
      RETURN NEW;
  END;
  $$ language 'plpgsql';

  CREATE TRIGGER update_users_cache_modtime
      BEFORE UPDATE ON users_cache
      FOR EACH ROW
      EXECUTE PROCEDURE update_modified_column();

  -- 論理削除されていないレコードへの高速アクセス用インデックス
  CREATE INDEX idx_users_cache_active
      ON users_cache(student_id)
      WHERE deleted_at IS NULL;
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `student_id` | 学籍番号 | VARCHAR(50) | PRIMARY KEY | - | 学生の学籍番号。主キー。 |
| `name` | 氏名 | VARCHAR(100) | NOT NULL | - | 学生の氏名。 |
| `department` | 学科 | VARCHAR(100) | NOT NULL | - | 所属学科。 |
| `grade` | 学年 | VARCHAR(20) | NOT NULL | - | 学年。 |
| `class_name` | クラス名 | VARCHAR(50) | NOT NULL | - | クラス名。 |
| `is_staff` | 教職員フラグ | BOOLEAN | NOT NULL | FALSE | 教職員の場合TRUE。学科・学年入力を省略するために参照する。 |
| `created_at` | 作成日時 | TIMESTAMPTZ | - | NOW() | レコードが最初に作成された日時。 |
| `updated_at` | 更新日時 | TIMESTAMPTZ | - | NOW() | レコードが最後に更新された日時。 |
| `deleted_at` | 論理削除日時 | TIMESTAMPTZ | - | NULL | 管理者が削除した日時。NULLの場合は有効なレコード。30日経過後に物理削除。 |

### 1.5 `usage_logs` テーブル定義
* **役割**: ジムの利用記録（チェックイン・チェックアウト）。オンライン時は直接インサートされ、オフライン時は復帰後にバルクインサートされます。
* **DDL（PostgreSQL）**:
  ```sql
  CREATE TABLE usage_logs (
      id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
      student_id VARCHAR(50) NOT NULL,
      name VARCHAR(100) NOT NULL,
      department VARCHAR(100) NOT NULL,
      grade VARCHAR(20) NOT NULL,
      class_name VARCHAR(50) NOT NULL,
      checked_in_at TIMESTAMPTZ NOT NULL,
      checked_out_at TIMESTAMPTZ DEFAULT NULL,
      created_at TIMESTAMPTZ DEFAULT NOW(),
      deleted_at TIMESTAMPTZ DEFAULT NULL
  );

  -- 検索高速化のためのインデックス
  CREATE INDEX idx_usage_logs_student_id ON usage_logs(student_id);
  CREATE INDEX idx_usage_logs_checked_in_at ON usage_logs(checked_in_at);

  -- 在室中（チェックアウト未記録）レコードの高速検索用インデックス
  CREATE INDEX idx_usage_logs_checkin_active
      ON usage_logs(student_id, checked_in_at)
      WHERE checked_out_at IS NULL AND deleted_at IS NULL;

  -- 論理削除されていないレコードへのインデックス（運用後の件数増加に応じて追加を検討）
  -- CREATE INDEX idx_usage_logs_active
  --     ON usage_logs(checked_in_at)
  --     WHERE deleted_at IS NULL;
  ```

#### カラム詳細
| 物理名 | 論理名 | データ型 | 制約 | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `id` | ログID | BIGINT | PRIMARY KEY | - | 自動連番のプライマリキー。 |
| `student_id` | 学籍番号 | VARCHAR(50) | NOT NULL | - | 入室した学生の学籍番号。 |
| `name` | 氏名 | VARCHAR(100) | NOT NULL | - | 入室した学生の氏名。 |
| `department` | 学科 | VARCHAR(100) | NOT NULL | - | 入室した学生の学科。 |
| `grade` | 学年 | VARCHAR(20) | NOT NULL | - | 入室した学生の学年。 |
| `class_name` | クラス名 | VARCHAR(50) | NOT NULL | - | 入室した学生のクラス名。 |
| `checked_in_at` | チェックイン時間 | TIMESTAMPTZ | NOT NULL | - | 利用者が「チェックイン」を押した（またはスキャンが成功した）時間。 |
| `checked_out_at` | チェックアウト時間 | TIMESTAMPTZ | - | NULL | 退室時間。NULLの場合は在室中または自動チェックアウト未処理。チェックイン後15時間経過で自動的にセットされる。 |
| `created_at` | レコード作成日時 | TIMESTAMPTZ | - | NOW() | データベースに書き込まれた日時（サーバー時間）。 |
| `deleted_at` | 論理削除日時 | TIMESTAMPTZ | - | NULL | 管理者が削除した日時。NULLの場合は有効なレコード。30日経過後に物理削除。 |

## 2. クライアントサイドデータベース（IndexedDB）定義

設置PCのブラウザがオフライン（インターネット未接続）状態のとき、一時的に利用ログを蓄積しておくためのローカル定義です。

* **データベース名**: `JimReserveOfflineStore`
* **ストア（テーブル）名**: `offline_logs`
* **キーパス (KeyPath)**: `id` (Auto Increment)

#### オブジェクトストア構造 (`offline_logs`)
| プロパティ名 | データ型 | 説明 |
| :--- | :--- | :--- |
| `id` | Number | ローカル自動連番（主キー） |
| `student_id` | String | 学籍番号 |
| `name` | String | 氏名 |
| `department` | String | 学科 |
| `grade` | String | 学年 |
| `class_name` | String | クラス名 |
| `checked_in_at` | String (ISO 8601) | 利用者がチェックインした日時 |
| `action` | String | `"checkin"` または `"checkout"` |
