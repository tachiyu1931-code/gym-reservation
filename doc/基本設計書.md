# ジム利用記録システム 基本設計書

## 1. システム概要
本システムは、ジムの入り口に設置したWindows PC（ブラウザ画面）を利用して、利用者が「入室記録」を簡便に登録するためのシステムです。
クラウドインフラとして **Vercel**（Webアプリケーションホスティング）と **Supabase**（クラウドデータベース）を採用し、設置端末にはサーバー環境やデータベース環境を一切構築することなく、Webブラウザの全画面表示のみで稼働します。

## 2. システムの特徴・設計方針
1. **クラウドネイティブかつゼロ・ローカル設定**:
   * ローカルPCにはNode.jsやSQLiteなどの環境構築が一切不要です。ブラウザでVercel上の特定URLを開くだけで即座に動作します。
2. **簡易な操作性とキャッシュによる入力支援（マスタ登録レス）**:
   * 初回利用時のみ「学科・学年・クラス・氏名・学籍番号」を手動入力します。
   * 2回目以降は、学籍番号の入力（または学生証スキャン）だけで、クラウド（Supabase）から取得した情報、またはブラウザ内にキャッシュされた情報が自動補完され、1タップで入室が完了します。
3. **ブラウザIndexedDBを活用したオフライン対応（耐障害性）**:
   * クラウド接続（インターネット）が一時的に切断された場合でも利用できるように、ブラウザ内のデータベース `IndexedDB` にログを一時保存します。
   * ネットワーク復旧（オンライン復帰）時に、バックグラウンドで未同期ログをSupabaseへと自動的に順次同期します。
4. **Basic認証によるシンプルなWeb管理画面**:
   * 管理画面（`/admin`）を同アプリ内に統合。
   * 開発・管理コスト削減のため、複雑な会員認証（ID/PasswordテーブルやOAuthなど）は導入せず、**Next.js Middleware** を使用したシンプルな **Basic認証** を採用します。
   * 管理者は任意の端末からURLにアクセスし、ブラウザ標準のダイアログに入力するだけで、リアルタイムで利用ログの閲覧、CSVダウンロード、学生キャッシュデータのメンテナンスが行えます。

## 3. 全体システムフロー

### 3.1 チェックインフロー（オンライン・通常時）
```mermaid
sequenceDiagram
    actor 利用者
    participant 画面 (ブラウザ)
    participant Next.js API (Vercel)
    participant PostgreSQL (Supabase)

    利用者->>画面: 学籍番号を入力
    画面->>Next.js API: キャッシュ検索リクエスト (学籍番号)
    Next.js API->>PostgreSQL: users_cache テーブル検索
    PostgreSQL-->>Next.js API: 結果返却
    Next.js API-->>画面: キャッシュデータ (氏名、クラス等)

    alt キャッシュが存在する場合 (2回目以降)
        画面->>画面: フォームに自動補完
        利用者->>画面: 内容を確認して「チェックイン」を押下
    else キャッシュが存在しない場合 (初回)
        利用者->>画面: クラス・氏名を手動入力して「チェックイン」を押下
    end

    画面->>Next.js API: チェックイン要求
    Next.js API->>PostgreSQL: usage_logs へのインサート & users_cache の UPSERT
    PostgreSQL-->>Next.js API: 成功
    Next.js API-->>画面: 登録完了
    画面-->>利用者: チェックイン完了表示 (3秒後に初期画面に戻る)
```

### 3.2 チェックインフロー（オフライン・一時切断時）
```mermaid
sequenceDiagram
    actor 利用者
    participant 画面 (ブラウザ)
    participant IndexedDB (ローカル)
    participant Next.js API (Vercel)

    利用者->>画面: 学籍番号を入力し、チェックインを実行
    画面->>Next.js API: APIリクエスト (HTTP POST)
    Note over 画面, Next.js API: ネットワーク切断によりタイムアウトまたはエラー
    
    画面->>画面: オフライン状態を認識
    画面->>IndexedDB: 利用ログを offline_logs ストアに保存
    画面-->>利用者: 「オフライン登録完了」を表示 (3秒後に初期画面へ)
    Note over 画面: 画面右下に「未送信ログ: 1件」と表示される
```

### 3.3 管理者ログインフロー（Basic認証）
```mermaid
sequenceDiagram
    actor 管理者
    participant 画面 (ブラウザ)
    participant Next.js Middleware (Vercel)
    participant 管理画面ダッシュボード (Next.js)

    管理者->>画面: /admin 以下のURLにアクセス
    画面->>Next.js Middleware: リクエスト送信 (認証ヘッダーなし)
    Next.js Middleware-->>画面: HTTP 401 Unauthorized (Basic認証要求)
    Note over 画面: ブラウザ標準のユーザー名/パスワード入力ダイアログを表示
    管理者->>画面: ユーザー名/パスワードを入力して送信
    画面->>Next.js Middleware: リクエスト送信 (Authorizationヘッダーあり)
    Next.js Middleware->>Next.js Middleware: 環境変数 (ADMIN_USER/ADMIN_PASSWORD) と照合
    Next.js Middleware-->>画面: HTTP 200 OK
    画面->>管理者: 管理画面ダッシュボードを表示
```
