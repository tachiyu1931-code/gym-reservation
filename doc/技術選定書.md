本システムの開発・運用にあたって採用する技術スタック、アーキテクチャ、およびその選定理由を記述します。

---

## 1. 技術スタック概要

| 区分 | 技術・サービス名 | 選定バージョン（目安） | 役割 |
| :--- | :--- | :--- | :--- |
| **フレームワーク** | **Next.js** (App Router) | v14+ / React v18+ | フロントエンドUI、APIルート（バックエンド処理）、ルーティング |
| **ホスティング** | **Vercel** | - | クラウドインフラ（静的配信 ＋ Serverless Functions）**【確定】** |
| **データベース** | **Supabase** (PostgreSQL) | - | クラウドデータベース（利用ログ、学生キャッシュ、マスタデータの保存） |
| **認証 (Auth)** | **Basic認証** (Next.js Middleware) | - | 管理者ページ（`/admin`）へのアクセス制限。資格情報はVercel環境変数で管理。 |
| **OCRライブラリ** | Tesseract.js | v4.x | ブラウザ上での学生証テキスト認識（フェーズ2） |
| **ローカル一時保存** | **IndexedDB** | ブラウザ標準 | オフライン時のログ一時保存（耐障害性） |
| **クライアントアプリ** | Google Chrome（キオスクモード） | 最新安定版 | 設置PCで全画面UIとして実行。`--kiosk` オプションで起動し、利用者による操作範囲を制限する。 |

## 2. 各技術の選定理由

### 2.1 Webフレームワーク（Next.js）
* **選定理由**:
  * **Vercelへのデプロイの親和性**: Vercelと完全に統合されており、パフォーマンスの最適化やサーバーレス関数の配置が自動で行われます。
  * **フロントエンド・バックエンドの一体化**: App Routerの API Routes や Server Actions を利用することで、フロントエンドとバックエンド（Supabaseとの通信や認証）を単一のコードベースでシームレスに記述できます。

### 2.2 ホスティングインフラ（Vercel）【確定】
* **採用決定**: 本システムのホスティング先はVercelに確定しました。
* **選定理由**:
  * サーバーの構築・管理（OSアップデートやミドルウェアのセットアップ）が一切不要な**サーバーレスインフラ**です。
  * Git（GitHub等）と連携することで、コードプッシュと同時に自動でビルド・デプロイが行われ、保守運用コストが極めて低くなります。
  * Next.jsとの親和性が最も高く、App RouterのServer ComponentsやServer Actionsが追加設定なしで最適動作します。
  * 今回の利用規模（約800名、月間数千件のログ）では無料枠（Hobby Plan）の範囲内で十分運用可能です。

### 2.3 クラウドデータベース（Supabase）
* **選定理由**:
  * **Firebase代替のオープンソースPaaS**: PostgreSQLをベースとしており、信頼性が高く、リレーショナルなデータ構造をクリーンに扱えます。
  * **高速なAPI生成**: テーブルを作成するだけで自動的にセキュアなREST APIが生成され、Next.jsからのデータ操作（CRUD）が容易になります。
  * **無料枠の充実**: 今回のジム利用記録（月間数千件程度）の規模であれば、十分に無料枠の範囲内で運用可能です。

### 2.4 認証（Basic認証）
* **選定理由**:
  * **最小限の開発・管理コスト**: ログイン用UI（画面）、セッション管理、認証情報のDBテーブルなどを構築する必要が一切ありません。
  * **Next.js Middlewareによる簡単実装**: `middleware.ts` を用いて、`/admin` 以下のパスにアクセスした際にブラウザ標準のBasic認証ダイアログを表示させ、Vercelの環境変数（`ADMIN_USER` / `ADMIN_PASSWORD`）に設定したID・パスワードと照合するだけでセキュアな領域を確保できます。
  * **利用者への露出なし**: 利用者（学生）がアクセスする `/` 以下のパスには一切認証がかかりません。

### 2.5 OCRライブラリ（Tesseract.js）
* **選定理由**:
  * 学生証の画像をクラウドに送信することなく、**設置PCのブラウザ内で完結してOCRを実行**できます。
  * 個人情報（顔写真等）をインターネット上に送信しないため、セキュリティ・個人情報保護の観点から極めて安全であり、かつVercelのサーバーレス実行時間制限（Timeout）の影響を受けません。

### 2.6 ローカル一時保存（IndexedDB）
* **選定理由**:
  * インターネット接続が一時的に切断された場合でもチェックイン・チェックアウト処理を継続するため、ブラウザ内の大容量ローカルストレージである IndexedDB にログを一時保存（キャッシュ）します。
  * ネットワーク復旧時に、自動的にVercel/Supabaseへ同期リクエストを送る「オフラインファースト」の動作を実現します。

### 2.7 クライアントアプリ（Google Chrome キオスクモード）
* **選定理由**:
  * **端末の安全な占有**: 利用者が設置PCのブラウザを閉じたり、他のアプリケーションを開いたりできないよう、Chromeのキオスクモードで起動します。
* **起動コマンド（Windows）**:
  ```bat
  chrome.exe --kiosk --disable-infobars --disable-session-crashed-bubble ^
             --noerrdialogs --start-fullscreen ^
             "https://your-app.vercel.app"
  ```
* **自動起動設定**: Windowsのタスクスケジューラまたはスタートアップフォルダにショートカットを登録し、PC起動時に自動的にキオスクモードで起動させます。
* **キオスクモード終了**: `Alt+F4` 等の標準ショートカットはChrome側で無効化されています。管理者がメンテナンスを行う場合は、`Ctrl+Alt+Delete` からタスクマネージャーを起動してChromeを終了します。
