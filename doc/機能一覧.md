# ジム利用記録システム 機能一覧

本システムが提供する機能を分類して定義します。認証方式にBasic認証を採用したクラウド構成に適合させています。

---

## 1. 利用者（学生）向け機能

利用者がジム入室時に操作、または視認する機能です。

| 機能ID | 機能名 | 機能概要 | 実装フェーズ |
| :--- | :--- | :--- | :--- |
| **U-01** | **初期（待受）画面表示** | 利用者を迎え入れる画面。画像認識（フェーズ2）または手動入力を促す。 | フェーズ1 |
| **U-02** | **学籍番号入力** | テンキーまたはキーボードから学籍番号を入力する。 | フェーズ1 |
| **U-03** | **キャッシュ自動補完** | 入力された学籍番号をもとにSupabase DBを検索し、一致するデータがあれば「氏名」「学科」「学年」「クラス」を自動入力する。 | フェーズ1 |
| **U-04** | **手動入力（初回用）** | キャッシュがない場合、利用者が「学科（選択）」「学年（選択）」「クラス（入力）」「氏名（入力）」を画面上で入力する。 | フェーズ1 |
| **U-05** | **チェックイン実行** | 入力データを確認し、チェックインボタンを押すことで利用ログを登録する。 | フェーズ1 |
| **U-06** | **完了フィードバック表示** | 「チェックイン完了」と氏名を画面に大きく表示し、3秒後に自動的に初期画面に戻る。 | フェーズ1 |
| **U-07** | **学生証カメラスキャン** | PCのカメラに学生証をかざすことで、映像からOCRで学籍番号を自動読取する。 | **フェーズ2** |
| **U-08** | **手動切り替え（フォールバック）** | スキャンがうまくいかない場合、または手動入力を選びたい場合に、1タップで手動入力画面に遷移する。 | **フェーズ2** |

## 2. 管理者向け機能

ジムの運営管理者が、Webブラウザを介してデータの確認やシステムの運用保守を行うための機能です。
※管理者はVercelでホストされた管理者用URL（例：`/admin`）にアクセスして操作します。

| 機能ID | 機能名 | 機能概要 | 実装フェーズ |
| :--- | :--- | :--- | :--- |
| **A-01** | **管理者ログイン認証（Basic）** | `/admin` アクセス時にブラウザ標準のダイアログを表示し、ID/PWを確認する。開発コストを削減するため、独自のログイン画面やセッション管理の実装は省略する。 | フェーズ1 |
| **A-02** | **利用ログ一覧表示** | これまでの利用ログを一覧で確認できる。日付範囲や学籍番号、氏名、クラス等で絞り込み・検索が可能。 | フェーズ1 |
| **A-03** | **利用ログCSVエクスポート** | 表示されている利用ログ、または指定期間の全利用ログをCSV形式でダウンロードする。 | フェーズ1 |
| **A-04** | **学生キャッシュ管理** | 登録されている学生キャッシュ（補完データ）の一覧を表示し、誤った情報の修正や不要なデータの削除を行う。 | フェーズ1 |
| **A-05** | **リアルタイム利用状況確認** | 本日の利用人数や、現在ジムにいると想定される人数（時間帯別統計など）をダッシュボード形式で確認できる。 | フェーズ1 |

## 3. システム共通・バックエンド機能

画面の裏側で動作する、データの永続化やネットワーク連携を行う機能です。

| 機能ID | 機能名 | 機能概要 | 実装フェーズ |
| :--- | :--- | :--- | :--- |
| **S-01** | **オンライン時直接登録** | ネットワーク接続がある場合、入力された利用ログを即時Supabase PostgreSQLに登録し、学生キャッシュをUPSERTする。 | フェーズ1 |
| **S-02** | **オフライン時一時保存** | ネットワーク切断時、フロントエンドで検知し、ログデータをブラウザの IndexedDB に一時保存する。 | フェーズ1 |
| **S-03** | **オンライン復帰検知と自動同期** | ブラウザがネットワーク復旧（`online` イベント）を検知すると、IndexedDB にある未同期ログをSupabaseに自動で順次同期する。 | フェーズ1 |
| **S-04** | **未同期ログの件数通知** | オフライン時に一時保存された未同期ログがブラウザ内に何件あるかを、チェックイン画面の片隅に小さく表示し、管理者や設置者が状況を把握できるようにする。 | フェーズ1 |
| **S-05** | **自動画面初期化（タイムアウト）** | 手動入力画面で一定時間（例：60秒）無操作だった場合、画面占有を防ぐために自動的に初期（待受）画面へ戻る。 | フェーズ1 |
